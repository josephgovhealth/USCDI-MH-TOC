NameCard.TypeConstraints and Usage
  Goal 0..* Goal dom-2: If the resource is contained in another resource, it SHALL NOT contain nested Resources
dom-3: If the resource is contained in another resource, it SHALL be referred to from elsewhere in the resource or SHALL refer to the containing resource
dom-4: If a resource is contained in another resource, it SHALL NOT have a meta.versionId or a meta.lastUpdated
dom-5: If a resource is contained in another resource, it SHALL NOT have a security label
dom-6: A resource should have narrative for robust management
Definition:
Describes the intended objective(s) for a patient, group or organization care, for example, weight loss, restoring an activity of daily living, obtaining herd immunity via immunization, meeting a process improvement objective, etc.


Comments:
Goal can be achieving a particular change or merely maintaining a current state or even slowing a decline.

  lifecycleStatus 1..1 code Binding:GoalLifecycleStatus (required)
ele-1: All FHIR elements must have a @value or children
Definition:
The state of the goal throughout its lifecycle.


Comments:
This element is labeled as a modifier because the lifecycleStatus contains codes that mark the resource as not currently valid.

  description 1..1 CodeableConcept Binding:US Core Goal Codes (preferred): Value set to describe the Goal


ele-1: All FHIR elements must have a @value or children
Definition:
Human-readable and/or coded description of a specific desired objective of care, such as "control blood pressure" or "negotiate an obstacle course" or "dance with child at wedding".


Comments:
If no code is available, use CodeableConcept.text.

  subject 1..1 Reference(US Core Patient Profile(8.0.0) S | Group | Organization) ele-1: All FHIR elements must have a @value or children
Definition:
Identifies the patient, group or organization for whom the goal is being established.

  start[x] 0..1 date S, CodeableConcept Binding:GoalStartEvent (example): Codes describing events that can trigger the initiation of a goal.


ele-1: All FHIR elements must have a @value or children
Definition:
The date or event after which the goal should begin being pursued.

  target 0..* BackboneElement ele-1: All FHIR elements must have a @value or children
gol-1: Goal.target.measure is required if Goal.target.detail is populated
Definition:
Indicates what should be done by when.


Comments:
When multiple targets are present for a single goal instance, all targets must be met for the overall goal to be met.

  due[x] 0..1 date S, Duration ele-1: All FHIR elements must have a @value or children
Definition:
Indicates either the date or the duration after start by which the goal should be met.

  expressedBy 0..1 Reference(US Core Practitioner Profile(8.0.0) S | US Core Patient Profile(8.0.0) | PractitionerRole | US Core RelatedPerson Profile(8.0.0)) ele-1: All FHIR elements must have a @value or children
Definition:
Indicates whose goal this is - patient goal, practitioner goal, etc.


Comments:
This is the individual responsible for establishing the goal, not necessarily who recorded it. (For that, use the Provenance resource.).
